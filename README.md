# SyncScribe — Real-time Collaborative Editor

프론트엔드 이직용 서브 프로젝트로 만든 **실시간 협업 에디터**입니다.  
핵심 목표는 아래 3가지를 실제 코드로 증명하는 것입니다.

- CRDT 기반 동시 편집 처리
- 협업 UX(커서/방 단위 세션)
- 실무형 제품 구조(로그인, 문서목록, 배포 준비)

## Stack

- React + TypeScript + Vite
- Tiptap Editor
- Yjs (CRDT)
- y-webrtc (P2P 실시간 동기화)
- y-indexeddb (오프라인 복구)
- LocalStorage 기반 간단 인증/문서 메타데이터 저장

## Implemented (Step 1 + Step 2)

### Step 1

- 실시간 공동 편집 (같은 Room ID 접속)
- 사용자별 협업 커서 표시
- IndexedDB 기반 오프라인 복구

### Step 2

- 로그인 화면 + 사용자 프로필 로컬 저장
- 문서 목록 사이드바 (생성/선택/삭제)
- 문서별 room 분리 협업
- 로그아웃/세션 상태 유지

## Architecture (요약)

- `문서 본문`: Yjs Doc + WebRTC provider로 동기화
- `문서 메타`: 로컬 스토리지(`title`, `updatedAt`, `id`) 관리
- `사용자 상태`: 로컬 스토리지(`name`, `color`) 관리
- `협업 세션`: 선택된 문서의 `id`를 room id로 사용

## Quick Start

```bash
npm install
npm run dev
```

동작 확인:

1. 브라우저 탭 2개를 연다.
2. 같은 문서를 선택한다.
3. 텍스트 입력 시 실시간 동기화되는지 확인한다.

### 4) `updatedAt` 저장 빈도 최적화

- 편집 이벤트 연속 발생 시 저장 빈도를 제어해 과도한 로컬 쓰기와 리렌더를 억제
- 문서 정렬 정확도는 유지하면서 체감 성능/배터리 효율을 개선

## Interview Highlights (문제-해결-성과)

1. **문제:** 브라우저 캐시/스토리지 손상 시 앱이 즉시 비정상 종료될 수 있었음  
   **해결:** 로컬 스토리지 파싱 예외 처리 + 안전한 fallback 초기화 로직을 추가  
   **성과:** 예외 상황에서도 핵심 기능(로그인/문서 진입)을 유지해 복구 탄력성 확보

2. **문제:** Room ID 공유가 환경 의존적(권한/브라우저 제약)이라 협업 시작 마찰이 발생  
   **해결:** Clipboard API 성공 플로우와 실패 대체 플로우(수동 복사 안내)를 모두 구현  
   **성과:** 협업 세션 진입 시간이 단축되고 "복사가 안 된다"는 사용자 이탈 포인트 감소

3. **문제:** 문서 삭제 시 선택 상태 혼선으로 오동작 리스크가 존재  
   **해결:** 삭제 확인 UX와 대상 식별 흐름을 분리해 의도하지 않은 삭제를 방지  
   **성과:** 데이터 안정성 신뢰도가 올라가고 문서 관리 작업의 심리적 부담 감소

4. **문제:** 편집 중 `updatedAt`를 과도하게 저장해 불필요한 렌더/쓰기 비용이 발생  
   **해결:** 저장 빈도 제어(디바운스/스로틀 기반)로 업데이트 밀도를 최적화  
   **성과:** 문서 목록 최신성은 유지하면서 성능 오버헤드를 줄여 실사용 반응성 향상

5. **문제:** 프로젝트 설명이 기능 나열 중심이라 면접에서 임팩트 전달이 약했음  
   **해결:** README를 문제-해결-성과 중심 서술로 재구성하고 주간 개선사항을 연결  
   **성과:** 포트폴리오 설명 시 기술 선택 근거와 사용자 가치 전달력이 함께 강화됨
