## 우선순위 (3~5개)

### P1. Playwright E2E 기본 시나리오 구축
- 내용: 로그인 → 문서 생성 → 문서 진입 → 편집 반영(로컬 저장 포함) 흐름을 자동 테스트
- 예상 난이도: **중(3/5)**
- 기대효과: **매우 높음(5/5)** — "작동한다"를 재현 가능한 형태로 증명, 포트폴리오 신뢰도 즉시 상승
- 소요시간(대략): **6~8시간**

### P2. 아키텍처 다이어그램 + 트러블슈팅 기록 문서화
- 내용: 현재 구조(클라이언트 상태/CRDT 동기화/문서 메타 저장) 시각화 + 최근 이슈 해결 과정 정리
- 예상 난이도: **하~중(2/5)**
- 기대효과: **높음(4/5)** — 면접 설명력 강화, 의사결정 근거 제시 가능
- 소요시간(대략): **3~5시간**

### P3. 문서 목록 서버 저장 + 권한 모델 최소 설계(Spike)
- 내용: Supabase 기준으로 문서 메타 테이블 스키마/권한 정책(RLS) 초안 및 클라이언트 연결 PoC
- 예상 난이도: **상(4/5)**
- 기대효과: **높음(4/5)** — 로컬 한계를 넘어 실서비스 구조로 확장하는 발판
- 소요시간(대략): **8~12시간**

### P4. 인증 고도화 방향 결정(Supabase Auth vs Auth.js) 및 적용 계획
- 내용: 현재 로컬 인증에서 마이그레이션 경로 비교표 작성, 다음 스프린트 적용안 확정
- 예상 난이도: **중(3/5)**
- 기대효과: **중~높음(3.5/5)** — 이후 서버 권한/협업 모델 구현 리스크 감소
- 소요시간(대략): **2~4시간**
